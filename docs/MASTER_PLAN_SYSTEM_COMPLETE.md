### **Structure:**
```
docs/
  MASTER_PLAN.md                    ‚Üê THE execution plan (single source of truth)
  MASTER_PLAN_TRACKER.md            ‚Üê Registry of all plans
  MASTER_PLAN_REQUIREMENTS.md       ‚Üê Specifications

    [plan_name]/
      [PLAN_NAME]_PLAN.md           ‚Üê Detailed implementation guide
      progress.md                   ‚Üê Detailed progress tracking
      status.md                     ‚Üê Current status
      blockers.md                   ‚Üê Blockers/dependencies
      working_status.md             ‚Üê Active work items
```

### **How It Works:**

1. **Master Plan** = Execution sequence (what to do, when)
2. **Individual Plans** = Reference guides (how to do it, details)
3. **Master Plan Tracker** = Registry (all plans, status, priorities)
4. **Cursor Rules** = Automation (new features auto-integrate)

- New features automatically integrate into Master Plan
- Optimal position always determined
- Catch-up logic automatically applied
- No manual integration needed
